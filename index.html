  • Angular Pages

    Angular-pages is a angular.js directive that lays out pages in rows and columns, and then allow users to transition between the pages, either by swiping or paging.

    Let me start by thanking @revolunet for his awesome angular-carousel directive. @revolunet has done an amazing job with it and if you are looking for a carousel tool, definitely check it out. Angular-pages is heavily inspired by and borrowed from angular-carousel.

    The navigation in the bottom right corner will show you which direction has more pages. You can click on the arrow to go there, swipe towards the opposition direction on your mobile device, or click-n-drag on your desktop browser.

  • Containers, Collections, Pages

    Angular-pages is fairly simple to setup. There are three layers to consider: container, collection and page. Container is the top level DIV element that contains multiple collections. A collection is a UL element containing multiple LI elements. Each LI element is a page.

    There can be multiple containers in a single HTML document. Each container can have multiple collections. And each collection can have multiple pages.

    Collections can contain a list of static pages (see top row of this demo), or it can be dynamic (see middle row). A dynamic collection means each page in the collection is generated by an ng-repeat directive.

    Collections can also be laid out in row or column format. By default, if not specified, the format is row. If "col" is specified, then column format is used.

  • Attributes

    zn-pages

    zn-pages is the main directive used to specify that this is a angular-pages container. By default the layout is by row. You can specify column layout by using zn-pages="col".

    zn-pages-start

    You can also indicate the start row and page with zn-pages-start attribute. This attribute is zero (0) based, so the first page is page 0, second page is page 1, etc. If this attribute appears in the div container, then it indicates the starting row/column. If it's part of the UL, then it indicates the starting page.

    zn-pages-swipe

    This attribute, when placed at the outer div, indicates whether to enable swiping action for this container or not.

  • znPageManager

    znPageManager is an angular service that provides several key functions so you can transition the pages with javascript.

  • {{ val }}

    This is an example page generated by an ng-repeat collection. There are 5 colors on the list, including ['#1abc9c', '#f1c40f', '#d35400', '#2c3e50', '#8e44ad'].

    Click on the following button to rotate the colors around. Each click will take the head of the color list and put it at the end of the list. This effectively changes the ng-repeat collection, and triggers angular-pages to update the list as well. However, you most likely won't notice any difference immediately unless you start moving around.
